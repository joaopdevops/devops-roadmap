## 🇬🇧 English

**Bridge into routing**
- Recapped the Month 01 path with the canonical "post office" family: 2.1 IP (apartment number) → 2.2 subnetting (which building) → 2.3 DHCP/DNS/NAT (the doorman dispatching) → 2.4 MTU (the truck's box). The missing piece: **how a packet gets from one network to ANOTHER = routing**.

**New concept — route decision (longest prefix match)**
- **Router = post office; routing table = the sorting board.** Each line is a destination rule.
- **Golden rule:** the **most specific** matching line wins (*longest prefix match*). The mask = how specific a rule is — `/24` specific; `/0` (`0.0.0.0/0` = default) matches everything but is the least specific → last resort.
- **Algorithm:** (1) read the destination IP; (2) of all matching lines, pick the longest prefix; (3) nothing specific matched → default; no default → drop + ICMP Destination Unreachable.

**Lab before theory — read my own `ip route`**
- `192.168.0.0/24 dev wlp0s20f3 proto kernel scope link` = **connected route** (home LAN); `proto kernel` = auto-created by the OS; `scope link` = reachable directly, no middleman.
- `default via 192.168.0.1 proto dhcp` = **default route** (`/0`); the `via` = hand it to the gateway; learned from the router via DHCP.
- `172.17.0.0/16 dev docker0 ... linkdown` = **connected route for Docker** (the `docker0` from Lab 2.4); `linkdown` = no container running.
- **Key trick:** a connected route has **no `via`** (deliver directly); the default **has `via`** (hand to the gateway). The presence of `via` tells you instantly whether a middleman is needed.
- **Longest prefix match live:** `ping 192.168.0.50` → matches the `/24` (specific) → delivered directly; `ping 8.8.8.8` → only matches the default (`/0`) → sent to the gateway.
- **No static route showed up** — normal on a PC (a static route is hand-configured on a router). That is the next step (Packet Tracer between two networks).

**Why it matters for DevOps**
- "Wrong route / missing route" is a top cause of "my container can't reach service X". Reading a routing table finds it in seconds.

## 🇧🇷 Português

**Ponte pro roteamento**
- Recapitulei o percurso do Mês 01 com a família canônica dos Correios: 2.1 IP (número do apto) → 2.2 subnetting (qual prédio) → 2.3 DHCP/DNS/NAT (o síndico despachando) → 2.4 MTU (a caixa do caminhão). A peça que faltava: **como um pacote vai de uma rede pra OUTRA = roteamento**.

**Conceito novo — decisão de rota (longest prefix match)**
- **Roteador = agência dos Correios; tabela de rotas = o painel de triagem.** Cada linha é uma regra de destino.
- **Regra de ouro:** ganha a linha **mais específica** que casa (*longest prefix match*). A máscara = o nível de especificidade — `/24` específica; `/0` (`0.0.0.0/0` = default) casa com tudo mas é a menos específica → último recurso.
- **Algoritmo:** (1) lê o IP de destino; (2) entre as linhas que casam, escolhe a de prefixo mais longo; (3) nada específico casou → default; sem default → descarta + ICMP Destination Unreachable.

**Lab antes de teoria — li o meu próprio `ip route`**
- `192.168.0.0/24 dev wlp0s20f3 proto kernel scope link` = **rota conectada** (LAN de casa); `proto kernel` = criada automática pelo SO; `scope link` = alcança direto, sem intermediário.
- `default via 192.168.0.1 proto dhcp` = **rota default** (`/0`); o `via` = passa pro gateway; ensinada pelo roteador via DHCP.
- `172.17.0.0/16 dev docker0 ... linkdown` = **rota conectada do Docker** (o `docker0` do Lab 2.4); `linkdown` = sem container rodando.
- **Truque de ouro:** rota conectada **não tem `via`** (entrega direto); a default **tem `via`** (passa pro gateway). A presença do `via` diz na hora se precisa de intermediário.
- **Longest prefix match ao vivo:** `ping 192.168.0.50` → casa o `/24` (específica) → entrega direta; `ping 8.8.8.8` → só casa a default (`/0`) → vai pro gateway.
- **Rota estática não apareceu** — normal num PC (estática é configurada na mão num roteador). É o próximo passo (Packet Tracer entre duas redes).

**Por que importa pra DevOps**
- "Rota errada / falta de rota" é causa nº1 de "meu container não alcança o serviço X". Ler uma tabela de rotas acha o problema em segundos.
